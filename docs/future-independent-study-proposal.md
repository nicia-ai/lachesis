# Proposal for a future independent study

Status: **proposal only; not registered, materialized, or authorized**.

Any future independent-author study should begin under a new external
registration and clean infrastructure. It must not amend, inherit, reuse, or
derive identities, corpora, packets, bindings, credentials, signing keys,
controllers, or operational claims from M7c.

## Entry conditions

- Mount no historical audit repository into any preparation, execution, or
  analysis environment.
- Create a minimal sanitized protocol capsule containing only the information
  each role requires.
- Enforce globally unique phase, attempt, task, reservation, and provider
  request identities before any artifact is sealed.
- Use a managed or independently operated credential and revocation service
  whose control plane is outside author, adjudicator, constructor, and analysis
  environments.
- Exercise the real controller entrypoint with positive non-study canaries
  before preregistration.
- Run a smaller, explicitly staged sample before requesting the full
  independent-author study.
- Complete an independent operational review before sealed corpus construction.

## Proposed stages

1. **Operational qualification.** Validate role images, real entrypoints,
   identity issuance, append-only records, lifecycle cleanup, and kill gates
   using only synthetic non-study data.
2. **Externally reviewed pilot registration.** Freeze a small sample, exact
   costs, prompts, package and documentation digests, strata, adjudication, and
   stopping rules.
3. **Staged pilot execution.** Execute only the registered pilot. Stop on any
   false equivalence, disclosure, binding drift, missing stratum, unsafe repair
   advice, or revocation failure.
4. **Independent operational review.** Review the pilot's isolation and record
   integrity without accessing hidden scientific labels.
5. **New full-study registration.** Only a separately justified full study may
   define a fresh population and corpus. Pilot results must remain separate and
   must not be pooled into confirmatory endpoints.

Human authors require their own unpooled protocol. Agent and human evidence must
remain separately sampled, adjudicated, analyzed, and reported.

This proposal creates no API, corpus, environment, credential, project change,
provider request, M8 work, or authorization text.
