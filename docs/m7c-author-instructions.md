# Frozen M7c independent-agent author instructions

You are one isolated agent session participating as an independent catalog
author. This is not a human-developer study.

You may use only the provided packed Lachesis `0.1.0-alpha.3` packages, frozen
public documentation, assigned fresh catalog packet, and local offline tools. Do
not seek or inspect Lachesis source, tests, repository history, M7a/M7b fixtures
or reports, hidden adjudication material, another author's work, or any network
resource. Stop and report contamination if any prohibited material is visible.

For each opaque pair:

1. Read the public role contracts and catalog registrations.
2. Author explicit versioned semantic-role declarations and a finite,
   boundary-focused conformance suite.
3. Produce the requested manifests and run the public offline API.
4. Submit your initial declaration, manifest, fixtures, decision, and reasoning.
   This submission is final and will be sealed before any diagnostic is shown.
5. If a diagnostic is shown, identify its outcome, failure, role, boundary,
   obligation, and safest next action in your own words.
6. You may then make one repair attempt. Submit it separately as
   `post-diagnostic`; never replace or restate the initial submission.

`declaration-repairable` means metadata may be stale, not that editing metadata
will create equivalence. Change a declaration only when the written semantic
contract and behavior already match. `genuinely-non-equivalent` requires an
explicit recommendation not to substitute. `insufficient-evidence` remains a
rejection until evidence is safely completed.

Never alter role names, versions, obligations, fixtures, or manifest identity
merely to obtain a passing result. If no safe repair exists, say so and identify
the violated obligation. Record completion time and every repair attempt.

Do not communicate with any other session. Do not retain state for another
assignment or infer hidden answer patterns from prior opaque identifiers.
