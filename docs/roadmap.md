# Lachesis roadmap

## Completed research milestones

- M1 established structurally reliable typed plan generation and then added
  trusted semantic obligations.
- M2 compared functional JSON IR with restricted capability TypeScript and
  closed as a valid formal failure with informative operational results.
- M3 compared lexical and graph-derived evidence views. It closed as a formal
  failure for universal graph superiority while establishing reliable typed
  validation and repair infrastructure.
- M4 completed provider-aware evidence compilation, deterministic provenance,
  and optional TypeGraph parity. It closed with mixed conclusions: the original
  adaptive policy was rejected, and the narrow exploratory candidate was too
  expensive to confirm under the frozen prospective design.

See [M4 results](m4-results.md) for the closure record and exact nonclaims.

## M5: production evidence runtime

M5a provides an entirely offline vertical slice of the cohesive public
evidence-runtime API. It composes the executable-plan boundary, lexical-default
evidence compilation, reduced oracle output, deterministic validation and
provenance, content-addressed record/replay, and optional TypeGraph storage. See
the [M5a runtime](m5a-evidence-runtime.md).

The product boundary is:

- lexical-by-default model-facing evidence;
- optional TypeGraph storage, history, and provenance;
- reduced oracle outputs containing answer values and supporting fact IDs;
- deterministic validation, citations, paths, and provenance construction;
- live, replay, mock, and recorded-effect interpreters; and
- explicit budgets, capability boundaries, and replayable run identities.

Evaluation should use fresh natural workloads and emphasize:

- end-to-end reliability;
- provenance completeness and inspectability;
- deterministic replay;
- resource and monetary budget enforcement; and
- developer experience.

M5 is not a graph-superiority study. Lexical evidence remains the production
default, TypeGraph remains optional infrastructure, and the M4 exploratory
candidate remains research-only and unconfirmed. M5b.1 is closed as
`complete-operational-pass`. The replacement four-record probe and the frozen
24-record natural repository-history pilot both passed; the original
permission-failed M5b.0 probe remains immutable and report-only. See
[M5 results](m5-results.md).

M5c prepares the first public alpha entirely offline. It adds a cohesive facade,
verified examples, strict package boundaries, private recording and SQLite
hardening, packed-consumer checks, and an explicit alpha stability policy. It
does not publish packages or authorize another provider call. See the
[public-alpha guide](public-alpha.md).

No M4 corpus or inspected result should be relabeled as fresh confirmation in a
future milestone.
