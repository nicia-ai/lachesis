# M8b.1 Stage 4 implementation hostile test matrix

This is a prospective acceptance matrix. Stage 4a executes none of these
semantic command paths.

| Area              | Hostile or boundary case                                                                                             | Required observation                                                                                                                              |
| ----------------- | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Grammar           | `--suite` without `--conformance-out`                                                                                | Exit `64`; no module execution, output, or report.                                                                                                |
| Grammar           | `--conformance-out` without `--suite`                                                                                | Exit `64`; structural Stage 3 behavior is unchanged.                                                                                              |
| Grammar           | Suite mode with `--report -`                                                                                         | Exit `64`; no output transaction starts.                                                                                                          |
| Grammar           | Duplicate/unknown flag or malformed suite locator                                                                    | Exit `64`; no command report.                                                                                                                     |
| Aliasing          | Suite/left/right source alias, cross-side source alias, or any artifact/source or artifact/artifact normalized alias | Reject before module execution or mutation. Same-side catalog/policy exports may share one module and execute it once.                            |
| Acquisition       | Suite source symlink, parent symlink, non-file, oversized file, growth, truncation, replacement, or identity drift   | Fail closed using Stage 2a classification; never diagnose acquired-but-unbound bytes.                                                             |
| Loading           | Generator absent in structural mode                                                                                  | Structural command succeeds without loading generator.                                                                                            |
| Loading           | Generator import attempted in suite mode only                                                                        | Exactly one dynamic import; a missing declared dependency is an internal failure (`70`), while interrupted source execution is incomplete (`23`). |
| Loading           | Mutation after source hash and before/after import or export lookup                                                  | Exit `23`; no semantic record or output commit.                                                                                                   |
| Suite validity    | Named export missing or unsupported shape                                                                            | Strict invalid suite (`20`).                                                                                                                      |
| Suite validity    | Unknown field or schema-invalid fixture                                                                              | `INVALID_SUITE`, exit `20`; diagnosis is not called.                                                                                              |
| Suite sufficiency | Valid suite omits a required role                                                                                    | One verified `INCOMPLETE_FIXTURE_EVIDENCE` diagnostic; exit `13`.                                                                                 |
| Suite sufficiency | Valid suite duplicates fixture evidence                                                                              | One verified `DUPLICATE_FIXTURE_EVIDENCE` diagnostic; exit `13`.                                                                                  |
| Suite sufficiency | Valid evidence cannot localize a safe result                                                                         | One verified insufficient-evidence diagnostic; exit `13`; rejection preserved.                                                                    |
| Native identity   | Conformant native report field or digest mutated                                                                     | Public verifier rejects; exit `22`; no artifact committed.                                                                                        |
| Native identity   | Rejected native diagnostic, action, evidence, or digest mutated                                                      | Public verifier rejects; exit `22`; no report accepted.                                                                                           |
| Input binding     | Native left/right fingerprint differs from regenerated manifest                                                      | Exit `22`.                                                                                                                                        |
| Input binding     | Native manifest digest or fixture digest differs from acquired input                                                 | Exit `22`.                                                                                                                                        |
| Cardinality       | Diagnosis returns zero or more than one assessment, or record duplicated                                             | Exit `70`; summaries are never patched to hide the error.                                                                                         |
| Conformance       | Exact finite conformance, equal policy                                                                               | One conformant record, one conformance artifact, exit `0`.                                                                                        |
| Policy            | Conformant suite plus capability difference                                                                          | Conformant record plus separate review record; exit `10`; policy warning remains.                                                                 |
| Policy            | Conformant suite plus budget difference                                                                              | Exit `10`; no compatibility or safe-migration wording.                                                                                            |
| Precedence        | Genuine non-equivalence plus policy difference                                                                       | `DO NOT SUBSTITUTE`; exit `12`, not `10`.                                                                                                         |
| Precedence        | Insufficient evidence plus policy difference                                                                         | Exit `13`, not `10`.                                                                                                                              |
| Precedence        | Declaration-repairable plus policy difference                                                                        | Conditional guidance; exit `11`, not `10`.                                                                                                        |
| Repair safety     | Declaration action is rendered as accepted                                                                           | Test must fail; only the initial rejected outcome may exist.                                                                                      |
| Repair safety     | Genuine difference lacks `do-not-substitute` or violated obligation                                                  | Strict report construction rejects.                                                                                                               |
| Artifact bytes    | Reordered accepted native fields                                                                                     | Canonical output and raw checksum are byte-identical.                                                                                             |
| Artifact bytes    | Missing LF, extra LF, or noncanonical JSON                                                                           | Verification mismatch (`22`).                                                                                                                     |
| Artifact binding  | Semantic digest matches but raw bytes differ                                                                         | Checksum mismatch (`22`).                                                                                                                         |
| Artifact binding  | Raw bytes match but artifact/native semantic identity differs                                                        | Identity mismatch (`22`).                                                                                                                         |
| Rejection output  | Native rejection with pre-existing conformance target                                                                | Target remains byte-identical; report contains embedded diagnostic and no conformance artifact entry.                                             |
| Transaction       | Conformant result with either output existing and no `--replace`                                                     | Exit `23`; neither target changes.                                                                                                                |
| Transaction       | Artifact staging write/flush/readback fails                                                                          | Exit `23`; report is not installed.                                                                                                               |
| Transaction       | Report staging write/flush/readback fails                                                                            | Exit `23`; conformance artifact is not installed.                                                                                                 |
| Transaction       | Parent or leaf replaced before either rename                                                                         | Exit `23`; rollback leaves no new commit marker.                                                                                                  |
| Transaction       | Artifact rename succeeds and report rename fails                                                                     | Artifact is removed/restored; verified rollback exits `23`.                                                                                       |
| Transaction       | Rollback remove/restore or verification fails                                                                        | Exit `70`; no alternate report path or retry.                                                                                                     |
| Transaction       | Any post-parse transaction failure before report commit                                                              | Exactly one deterministic failure report on stdout; no conformance artifact claim.                                                                |
| Transaction       | Crash after artifact rename but before report rename                                                                 | Artifact is uncommitted; detached verification rejects the absent/stale pair.                                                                     |
| Transaction       | Crash after report commit but before backup cleanup                                                                  | Matching finals verify; stale private rollback material conveys no command outcome and is cleaned on controlled recovery.                         |
| Replace           | One or both prior targets replaced successfully                                                                      | New pair commits together logically; old/new cross-pairs fail checksum verification.                                                              |
| Paths             | Leaf or parent symlink, normalized escape, or output/source alias                                                    | Reject before mutation using Stage 2a machinery.                                                                                                  |
| Determinism       | Repeated run over identical bytes                                                                                    | Native artifact and command report are byte-identical.                                                                                            |
| Redaction         | Absolute path, environment-shaped text, ANSI, control, or secret in failures                                         | Deterministic redacted report; human output safely escaped.                                                                                       |
| Nonclaims         | Rendering contains equivalence, substitutability, safe migration, or generalized compatibility claims                | Test fails unless text is an explicit negation/nonclaim.                                                                                          |
| Historical        | Complete Stage 3 suite and immutable checksum audit                                                                  | Unchanged and passing.                                                                                                                            |
| Packaging         | Fresh workspace-free packed consumer, generator selected only by suite mode                                          | Node 24 ESM and strict TypeScript 6 pass; no public export added.                                                                                 |

Implementation is GO only when every applicable row passes together with the
complete repository verification matrix.
