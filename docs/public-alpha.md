# Public alpha guide

Lachesis `0.1.0-alpha.1` exposes one supported product path:

```text
compile a typed plan
  -> configure a pinned evidence store and trusted policy
  -> run an injected bounded oracle
  -> inspect the typed answer, citations, and provenance
  -> record a content-addressed artifact
  -> replay with zero oracle or evidence-store access
```

The alpha is ESM-only. It targets Node 24. The portable kernel, evidence
runtime, and host-provided TypeGraph adapter also bundle for Cloudflare Workers.
No public package reads credentials or dispatches a provider call on import.

The benchmark controller, campaign ledgers, research scorers, frozen fixtures,
provider credentials, and live-provider adapter package are private workspace
tools. They are not re-exported by the public runtime.

The exact export-by-export classification, counts, and content digests are
recorded in [`public-api-inventory.json`](public-api-inventory.json) and
regenerated by `pnpm audit:public-api`.

## Quickstart

Install the public runtime and Zod:

```bash
pnpm add @nicia-ai/lachesis-runtime@0.1.0-alpha.1 zod@4.4.3
```

Define nominal request and answer schemas, register a bounded oracle effect,
compile the JSON plan, and bind the returned plan and semantic-contract hashes
into a trusted policy. Supply an evidence store, an injected oracle effect, and
a recording store to `run()`. Always branch on `result.ok`.

The complete offline examples in [`examples/m5-alpha`](../examples/m5-alpha)
cover:

- in-memory mock execution;
- recording an injected oracle and exact replay;
- private-file recording and exact replay;
- a host-provided TypeGraph `HistoryStore`;
- managed private SQLite;
- historical snapshots, later retraction, and historical replay; and
- exhaustive typed failure handling.

`pnpm test:examples` runs every example without network access or credentials.

## API reference

### Compilation

- `compilePlan(planJson, catalog, policy, obligations)` returns an opaque
  `ExecutablePlan` only after schema, graph, policy, bounds, and semantic
  obligations pass.
- `createCatalog()` and the `define*()` helpers register nominal schemas and
  bounded operations.
- `inspectExecutablePlan()` exposes immutable identities and analyzed bounds,
  never the internal executable representation.

### Evidence and oracle setup

- `createInMemoryEvidenceStore()` creates deterministic recorded-time snapshots
  for tests, Workers, and host-managed data loading.
- `createOracleEffectIdentity()` binds the reduced request and answer schemas.
- `createMockOracleInterpreter()` resolves exact offline fixtures.
- `createRecordingOracleInterpreter()` wraps a host-injected bounded effect.
- `createMemoryRecordingStore()` provides an ephemeral recording store.
- `createPrivateFileRecordingStore()` from the Node subpath provides durable,
  private, content-addressed recording files.

### Run and replay

- `run()` validates inputs, the plan and semantic contract, pins the requested
  evidence snapshot, compiles lexical evidence by default, invokes the bounded
  oracle, validates only visible evidence, constructs citations and canonical
  provenance, saves a replay artifact, and returns its digest.
- `replay()` verifies every recorded identity, replays the plan, and
  reconstructs provenance without an evidence store or oracle effect.

`RuntimeResult` includes answer, citations, provenance, plan identity,
semantic-contract identity, evidence snapshot, visible-view identity, effect
identity, plan and oracle budgets, usage, and a redaction-safe trace.

## Runtime and effect lifecycle

1. The host compiles once and retains the opaque executable.
2. The host constructs a trusted policy from the inspected plan identity.
3. Each run pins valid-time and recorded-time coordinates before selection.
4. The lexical-default compiler produces a bounded normalized visible view.
5. The runtime passes only the public instruction, public answer contract, and
   normalized visible evidence to the oracle.
6. The runtime validates the adapter `Result`, usage, wire output, semantic
   obligations, citations, and provenance in distinct stages.
7. The recording store commits one canonical artifact by digest.
8. Replay requires the exact plan, task, policy, artifact, request, effect,
   snapshot, visible-view, output, result, and reconstruction identities.

The oracle receives an abort signal with the trusted wall-clock deadline. Caller
cancellation and deadline expiry race the effect even when an adapter does not
cooperate. Calls, tokens, wall time, and concurrency remain explicit trusted
limits.

## Trust boundaries

Lachesis preserves three different graphs:

1. the plan/orchestration graph in the opaque `ExecutablePlan`;
2. the knowledge/evidence graph supplied by the evidence store; and
3. the run/provenance graph reconstructed from accepted visible support.

Evidence topology never becomes executable control flow. Validation, citations,
canonical paths, and provenance use only the public task contract, the exact
model-visible evidence view, and the model's answer values and supporting fact
IDs. Source graphs remain selector/audit inputs only.

Traces contain bounded stage names and content digests. They do not contain
credentials, prompts, raw evidence, raw answers, exception messages, or hidden
evaluation values.

## Record and replay

Memory recording is useful for tests. The Node file store creates a dedicated
`0700` directory and exclusively writes `0600` temporary files. It fsyncs the
artifact, commits it with an atomic hard link, fsyncs the directory, and removes
the temporary file. Concurrent identical saves converge on the same file;
conflicting content fails closed. The design uses no lock file, so there is no
ambiguous lock-owner recovery. Stale private temporary files are recovered only
after their configured age.

Concurrent identical runs converge on one immutable recording artifact, but the
alpha runtime does not suppress duplicate external oracle invocation. Hosts that
inject live effects remain responsible for request-level idempotency or
coordination.

Loads reject traversal, malformed digests, symlinks, non-regular files,
ownership mismatch, permission drift, oversized artifacts, invalid JSON, and
schema drift. The portable runtime revalidates the artifact and every nested
identity after loading it.

## TypeGraph integration

The portable TypeGraph root accepts a host-provided TypeGraph 0.38
`HistoryStore`. It snapshots the knowledge graph into the same canonical
Lachesis evidence contract used by the in-memory store. Storage audit identities
may differ; model-visible views, semantic decisions, citations, paths,
provenance, and reconstruction identities must not.

The Node-only SQLite subpath pre-creates a dedicated database as `0600` inside a
`0700` directory without following the final symlink, audits ownership and mode
before TypeGraph opens it, and audits the database plus journal, WAL, and
shared-memory sidecars after opening. Permission drift fails closed.

TypeGraph remains additive infrastructure. It is not required by the runtime and
receives no model-quality credit.

## Alpha stability policy

Stable-alpha entrypoints keep documented export names and behavior compatible
within the `0.1` alpha line unless a security or correctness defect requires a
break. Such corrections receive a changelog entry and a new prerelease version.
Experimental entrypoints may change between alpha releases. Private workspace
packages carry no public compatibility promise.

Replay protocols and content identities are stricter than API compatibility: an
identity-changing correction never silently reinterprets an older artifact.

## Security guidance

- Treat plans, oracle output, evidence, and replay files as untrusted input.
- Keep capabilities and every budget in trusted host configuration.
- Use a dedicated private directory for SQLite and recording files.
- Audit permissions before dispatch and after execution when injecting a live
  provider outside the portable runtime.
- Never include credentials, hidden answers, or raw provider errors in traces.
- Reject snapshot, schema, adapter, and replay identity drift.
- Propagate one caller-owned abort signal through stores and effects.

This alpha is not a security certification. The private Node stores currently
require POSIX ownership/mode semantics and do not claim Windows support.

TypeGraph 0.38 itself declares `drizzle-orm` as a non-optional peer even though
the portable Lachesis adapter neither imports nor exposes Drizzle. Package
managers that auto-install peers may therefore install Drizzle transitively.
This is an upstream package-metadata limitation; the declaration audit proves
that it does not enter Lachesis's portable public types.

## Explicit nonclaims

The public alpha does not claim provider, graph, graph-serialization, TypeGraph,
or model-quality superiority. It does not claim production-scale performance,
availability, browser support, Windows private-storage support, or independent
security certification. No example, import, or package installation authorizes
provider dispatch.
